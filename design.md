# iSales Mobile Design System

## 视觉方向

iSales Mobile 采用高亮蓝白企业移动端风格：页面要像 iOS 原生工作台，信息密度高但层次轻，重点操作用蓝色强调，业务状态用绿色、橙色、红色区分。

## 色彩令牌

- 主色：`#0B6BFF`
- 主色深色：`#0057E7`
- 主色浅底：`#EAF2FF`
- 页面背景：`#F7FAFF`
- 卡片背景：`#FFFFFF`
- 信息块浅底：`#F3F6FC`
- 主文字：`#121826`
- 次文字：`#667085`
- 辅助文字：`#98A2B3`
- 分割线：`#E6EBF2`
- 成功：`#16B45F`
- 警示：`#FF8A1F`
- 风险：`#F53F3F`
- 紫色强调：`#6147FF`

## 字体与层级

- 字体族：`PingFang SC`, `Huawei Sans`, `Microsoft YaHei`, system-ui。
- 页面标题：24px / 600，用于顶部居中标题。
- 区块标题：20px / 700，用于卡片标题。
- 数据数字：24px / 700，用于指标卡核心值。
- 正文信息：13-16px，根据容器密度调整。
- 文案不使用负字距，中文保持自然字距。

## 间距与圆角

- 手机画布：430 x 932 基准，内容左右边距 15-16px。
- 区块间距：10-14px。
- 卡片圆角：18px。
- Banner 圆角：14px。
- 指标块圆角：12px。
- 快捷入口图标底圆角：13px。
- 项目行图片圆角：7px。
- 底部弹层圆角：20px。

## 阴影

- 页面卡片：`0 8px 24px rgba(21,51,99,.08)`。
- 重点蓝色按钮：`0 9px 18px rgba(11,107,255,.26)`。
- 悬浮新增按钮：`0 14px 24px rgba(11,107,255,.32)`。
- 弹层：`0 12px 34px rgba(32,56,96,.10)`。

## 核心组件

- 顶部栏：包含 iOS 状态栏、左侧圆形头像、居中页面标题、右侧更多按钮。
- 搜索栏：浅灰输入框加蓝色方形搜索按钮，默认文案为“搜索项目 / 客户 / 合同 / 回款单”。
- 主视觉 Banner：蓝色渐变背景，左侧标题与业务说明，右侧使用手机/数据卡片视觉元素。
- 系统公告：浅蓝灰胶囊条，左侧蓝色标题，右侧蓝色动作入口。
- 今日指标：3 列 x 2 行，每块包含标题、数值、趋势和圆形图标。
- 快捷入口：5 列 x 2 行，图标统一蓝色，底色为浅蓝灰渐变。
- 重点项目：缩略图、标题、客户、负责人、阶段、预计交付、状态标签和进度操作。
- 底部导航：5 个入口，当前项主蓝色，未选中为灰色。
- FAB：右下角蓝色圆形新增按钮，悬浮于重点项目和底部导航之间。
- 业务弹层：白色圆角底部 Sheet，承载更新进度、上报风险、审批、签字等操作。

## 交互规则

- 所有业务卡片点击后进入对应详情页，不使用通用假弹窗。
- 每个详情页必须展示“业务流程”，并标出已完成、进行中、待办、未开始等阶段状态。
- 阶段进展必须包含当前阶段名称、整体百分比、阶段说明和状态标签。
- 详情页底部动作最多展示 3 个，更多操作收进“•••”菜单。
- 更新、上报、签字、提交类操作必须使用明确的业务表单或确认弹层。
- Toast 只用于提交成功、同步完成、草稿保存等短反馈。
- 后续新增组件优先复用 `card`、`metric`、`quick`、`project`、`sheet`、`bottom-actions` 这些结构。

## 数据保存规则

- 原型内所有保存、提交、同步、上报动作都必须写入 `localStorage.isalesMobile.records`。
- 如果浏览器限制本地存储，则写入当前会话备份 `window.__isalesRecords`。
- 详情页必须展示“保存位置”卡片，说明业务对象、保存模块、原型存储位置和记录表。
- 最近 3 条操作记录必须在详情页实时显示，包含操作结果、时间、保存模块和记录表。
- 后端接入时，`localStorage.isalesMobile.records` 对应服务端操作流水；`保存到` 对应业务模块路由；`记录表` 对应业务库表或聚合视图。

## 禁用项

- 不使用大面积单色背景。
- 不使用过重投影。
- 不在卡片内再嵌套完整卡片。
- 不把说明性长文放在首页首屏。
- 不使用和主色冲突的紫蓝渐变作为主背景。
